The user interface was retained from the legacy system because it was sufficiently flexible and expressive enough to meet the query needs in most types of speech science research. The parser implemented in is based on the formal language definition of , which defines the symbols and the relationship of those symbols to each other on which this language is built (see adapted version of entire in Appendix ). Here we will describe the various terms and components that comprise the slightly adapted version 2 of the . It is worth noting that the new query mechanism uses a relational back-end to handle the various query operations (see Chapter for details). This means that expert users, who are proficient in may also query this relational back-end directly. However, we feel the provides a simple abstraction layer which is sufficient for most speech and spoken language research.

The most basic form of an query is a simple equality, inequality, matching or non-matching query, two of which are displayed in R Example . The syntax of a simple query term is , where specifies a level (or alternatively the name of a parallel attribute definition); is one of (equality), (inequality), (matching) or (non-matching); and is an expression specifying the labels of the annotation items of . The second query in R Example queries an event level. The result of querying an event level contains the same information as that of a segment level query except that the derived end times have the value zero.
