The function converts a TextGrid collection to the format. A precondition that all files have to fulfill is that they must all contain the same tiers. If this is not the case, yet there is an equal tier subset that is contained in all the TextGrid files, this equal subset may be chosen. For example, if all files contain only the tier the conversion will work. However, if a single of the collection has the additional tier the conversion will fail. In this case the conversion could be made to work by specifying the equal subset (e.g., ) and passing it on to the function argument . As can be seen in Figure , the TextGrid files provided by the demo data contain eleven tiers. To reduce the complexity of the annotations for this tutorial we will only convert the tiers (orthographic word annotations), (strong: vs. weak: syllable annotations), (phoneme level annotations) and (phonetic annotations) utilizing the parameter. This conversion can be seen in R Example .

<>= # convert TextGrid collection to the emuDB format convert_TextGridCollection(dir = tgColDir, dbName = “myFirst”, targetDir = tempdir(), tierNames = c(“Text”, “Syllable”, “Phoneme”, “Phonetic”)) @

The above call to creates a new directory in the directory called . This contains annotation files that contain the same , , and segment tiers as the original files as well as copies of the original () audio files. For further details about the structure of an , see Chapter of this document.
